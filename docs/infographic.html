그룹웨어 알림봇

그룹웨어 새 게시물을 실시간으로 감지해
디스코드로 즉시 알림을 전달하는 자동화 스크립트

Why
왜 만들었나요?
SI 프로젝트를 수행하다 보면 폐쇄망에서 근무하는 작업환경 때문에 그룹웨어 로그인이 뜸해지기 쉽습니다.
그러다 공지사항·인사발령·경조사 같은 중요한 게시물을 뒤늦게 발견하는 일이 반복됐습니다.

매번 직접 로그인하지 않아도 새 글이 올라오는 즉시 알림을 받을 수 있도록 이 봇을 만들게 됐습니다.

카카오톡 나와의채팅

카카오 OAuth를 연동해 '나에게 보내기' API로 알림을 전송하는 방식을 선택했습니다.

⚠️ 문제 발생 — '나와의채팅'은 앱 알림이 별도 설정 없이는 도착하지 않아, 정작 중요한 순간에 알림을 놓치는 상황이 반복됐습니다.

Development
AI 에이전트 팀으로 개발했습니다

이 봇은 Claude Code의 다중 에이전트 하네스를 구성해 개발했습니다.
사람이 직접 코드를 작성하는 대신, 분석가·엔지니어·테스터 역할을 맡은 세 AI 에이전트가 파이프라인을 이루어 기능을 설계·구현·검증합니다.

STEP 1 · ANALYST
🔍

분석가 에이전트

요구사항을 받아 코드베이스를 분석하고, 구현 스펙과 엣지 케이스를 문서로 정의합니다.

STEP 2 · ENGINEER
⚙️

엔지니어 에이전트

분석가가 작성한 스펙을 기반으로 Python 코드를 작성하고 파일을 수정합니다.

STEP 3 · TESTER
✅

테스터 에이전트

변경 사항에 대한 단위 테스트를 작성하고 실행해 기능이 올바르게 동작하는지 검증합니다.

🔍
ANALYST

분석가 — 요구사항 분석 & 스펙 정의

  • 코드베이스 전체를 탐색해 변경이 필요한 파일·함수를 특정
  • 엣지 케이스, 상태 관리 방식, 하위 호환성 영향 범위 검토
  • 엔지니어가 즉시 구현에 착수할 수 있는 스펙 문서 작성
📄 _workspace/01_analyst_spec.md 작성 후 SendMessage
↓ SendMessage → 엔지니어에게 스펙 전달
⚙️
ENGINEER

엔지니어 — Python 코드 구현

  • 스펙 문서를 읽고 main.py · notifier.py 등 실제 파일 수정
  • 상태 필드 추가, 로직 분기, 예외 처리 등 구현
  • 변경 파일 목록과 설계 결정 사항을 구현 노트로 기록
📄 _workspace/02_engineer_notes.md 작성 후 SendMessage
↓ SendMessage → 테스터에게 검증 요청
✅
TESTER

테스터 — 단위 테스트 작성 & 실행

  • 정상 흐름 / 경계 조건 / 예외 상황별 pytest 테스트 케이스 추가
  • pytest 실행 — 전체 15개 테스트 통과 여부 확인
  • 실패 시 원인을 엔지니어에게 SendMessage로 전달, 수정 후 재검증
📄 tests/test_main.py  →  pytest 실행 → 오케스트레이터에 결과 보고
⟳ 자동 피드백 루프 — 테스트 실패 시 테스터가 실패 원인을 엔지니어에게 SendMessage로 전달합니다. 엔지니어가 코드를 수정하면 테스터가 재검증하며, 최대 2회까지 자동으로 반복됩니다. 2회 후에도 해결되지 않으면 오케스트레이터(메인 에이전트)에게 에스컬레이션합니다.

Oracle Cloud 무료 VM

별도 비용 없이 OCI(Oracle Cloud Infrastructure) 무료 티어 VM을 생성해 24시간 상시 운영합니다.
Always Free 혜택으로 서버비 0원에 안정적으로 운영 중입니다.

SCHEDULER · CRON
⏰

Linux Cron 자동 실행

crontab에 두 가지 스케줄을 등록해 완전 자동으로 실행합니다.

# 새 게시물 알림 — 평일 08~20시, 5분마다
*/5 8-20 * * 1-5  python3 main.py
# 식단 알림 — 평일 오전 10시
0 10 * * 1-5    python3 main.py --meal
🚀
무료로 24시간 운영 중 — OCI Always Free VM + Linux Cron 조합으로 별도 서버 비용 없이 평일 업무 시간 내 5분 간격으로 그룹웨어를 모니터링합니다.
